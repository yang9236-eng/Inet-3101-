Problem Statement

This program simulates a seating reservation system for Colossus Airlines with 48 seats per flight and two flights (outbound and inbound).

Solution Description

The program uses an array of structures to represent seats. Each seat stores its ID, assignment status, and passenger name. A two-level menu allows users to manage seat reservations.

Pros
Simple structure-based design
Easy to extend for more flights
Clear menu-driven interaction
Cons
No persistent storage (data resets on restart)
No input validation for string overflow
Not optimized for large-scale systems
